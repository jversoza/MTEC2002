Git

We'll be using a version control system called git. For this class, we'll primarily use it as a way of uploading files to github (though - github is distinct from git).

At the beginning of each class:

We'll create a directory where you'll be doing your work. Then, we'll download the repository for your assignments. Finally, there are some minor configurations that have to be made - your username and password.
  1. open terminal
  2. change directory to Desktop {% highlight bash %} cd Desktop {% endhighlight %}
  3. make a directory with your first initial and last name {% highlight bash %} mkdir jversoza {% endhighlight %}
  4. change directory to the one that you just created {% highlight bash %} cd jversoza {% endhighlight %}
  5. log into github
  6. get the http URL of your repository
  7. make sure you're in the directory you just created {% highlight bash %} pwd # should return /Users/joe/Desktop/jversoza/ {% endhighlight %}
  8. download your assignments repository from github using the url that you copied above {% highlight bash %} git clone https://your_usrname@github.com/your_username/mtec2002_assignments.git {% endhighlight %}
  9. change directory to the repository that you just checked out {% highlight bash %} cd mtech2002_assignments {% endhighlight %}
  10. set your username (first initial, last name works) {% highlight bash %} # Note that this info is publicly available, so make sure you use a user name you're available git config user.name "your_username" {% endhighlight %}
  11. set your email {% highlight bash %} # Note that this info is publicly available, so take care to use an appropriate email address! git config user.email "your_email@citytech.cuny.edu" {% endhighlight %}
  12. Download the class zip files and extract them into the directory you're in (ex /Users/joe/Desktop/jversoza/mtec2002_assignments)
  13. change directory to the one that you just made {% highlight bash %} cd class6 {% endhighlight %}
  14. open that folder in finder

Committing ("Saving") /your Work:

  1. check to see if there's anything to commit or if there's anything staged to be committed {% highlight bash %} git status {% endhighlight %} you should see files that are not yet "tracked" by git under "Untracked files:" {% highlight bash %} # On branch master # Untracked files: # (use "git add ..." to include in what will be committed) # # class5/ {% endhighlight %}
  2. Stage your commit by adding files; check the status again {% highlight bash %} git add class5 git status {% endhighlight %} {% highlight bash %} # On branch master # Changes to be committed: # (use "git reset HEAD ..." to unstage) # # new file: class5/lists_loops_review.py # {% endhighlight %}
  3. Commit your work - use a relevant commit message after the "-m"! {% highlight bash %} git commit -m "put in a commit message here" {% endhighlight %} You should see something like... {% highlight bash %} [master 01f5b6e] added another line to... {% endhighlight %}
