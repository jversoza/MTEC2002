Getting started

We'll start our series of lab exercises by:

  • Installing tools/applications/software that we'll need for the class
  • Testing the installations
  • Doing the first exercise and sharing it online

Tools

  • Python (already installed on lab computers!)
  • A text editor - TextWrangler
  • Source code management - git (will be installed on lab computers)
  • A way to share code online - github.com
  • Obviously Python
  • Python is just text... so you need something to edit text. Feel free to use another text editor if it's already installed.
  • Source code management/version control system - absolutely necessary! No more .bak files, easy collaboration, branching code
  • github is website built on top of the version control system we're using, git. It facilitates code sharing, you'll see in a bit.

Python Install

The current 2.x release is 2.7.2, so we'll try to use a version as close to that as possible. This lab has 2.6.1 installed. At home...

  • Python is installed by default on OSX or Ubuntu
  • For Windows (XP and later) download an installer from python.org
  • Although Python 3.x is the eventual future of Python, it hasn't been widely adopted in production systems yet, so we're going with 2.x
  • 2.7.1 is on Lion and Natty
  • Though for most of what we're doing, 2.7.x or 2.6.x should suffice

There are a couple of ways to run Python code. Using the interactive shell, you can get immediate feedback on Python commands. The other way is to run a file using the intepreter... which we'll get to in a moment.

TextWrangler install

  • Download TextWrangler
  • Double click on the .dmg
  • Drag the app to the Applications folder... (or the desktop if you're prompted for a password - we'll clean it up later)
  • Double click on the TextWrangler icon to start it

Github

  • Create a github account by going to github.com
  • Go to the repository that has this course's assignment templates: https://github.com/jversoza/mtec2002_assignments
  • Fork this repository!
  • Check your newly forked repository
  • Again, github is a site that lets you share code online; it's built on top of the version control system that we'll be using, git
  • I created this repository to set up a template that can be followed for submitting labs/assignments
  • Forking a repository means making a copy of it so that you can edit it on your own. More generally, though, it means a divergent branch of the same code base that's made from a legal copy.

Putting it all together

Before starting the exercise, let's see how everything works together.

  • Create a file called lpthw_ex1.py in TextWrangler
  • Open terminal and find the file using the following commands:
    • pwd - print the name of the current working directory
    • ls - list the contents of a directory
    • cd [dir] - change to directory [dir]
    • cat [file ...] - print out the contents of [file]
  • use the python interpreter to run the program
  • python lpthw_ex1.py
  • (won't print anything yet...)
  • The path to the file is on top of TextWrangler
  • ~ means home directory
  • Talk about pwd, ls and cd
  • One way to run Python programs is to use the Python interpreter
